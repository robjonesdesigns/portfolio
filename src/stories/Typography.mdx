## Type tokens and sets

The portfolio uses type tokens across two modes to manage typography. Type tokens are pre-set configurations of typographic elements — font size, weight, line-height, and letter-spacing — calibrated for their role.

**Fluid (display)** — Four display tokens use `clamp()` so they scale smoothly between viewport sizes. No breakpoints needed.

**Fixed (body + labels)** — Body and label sizes are set in `px`. Reading comfort at these sizes depends on consistent line-height, not fluid scaling.

## Display set — fluid

All four display tokens are fluid. They scale continuously between a minimum and maximum size using `clamp()`. Line-height and letter-spacing are bundled into the token — applying `.text-display-*` sets all three values at once.

Font-weight is kept **separate and explicit** (`font-bold`) so semantic weight decisions remain in the component, not the token.

## How display tokens work

Display tokens are **not** generated by Tailwind JIT. The class names (`.text-display-2xl` etc.) contain hyphens that cause Tailwind's scanner to break them into sub-tokens it cannot resolve.

They are defined manually in `globals.css` inside `@layer utilities`:

```css
/* globals.css — @layer utilities */
.text-display-2xl { font-size: var(--text-display-2xl); line-height: 1;    letter-spacing: -0.04em; }
.text-display-xl  { font-size: var(--text-display-xl);  line-height: 1;    letter-spacing: -0.03em; }
.text-display-lg  { font-size: var(--text-display-lg);  line-height: 1.1;  letter-spacing: -0.03em; }
.text-display-md  { font-size: var(--text-display-md);  line-height: 1.25; letter-spacing: -0.02em; }
```

The CSS custom properties are defined in `:root`:

```css
:root {
  --text-display-2xl: clamp(3rem, 8vw, 6rem);      /* 48–96px */
  --text-display-xl:  clamp(2.5rem, 6vw, 5rem);    /* 40–80px */
  --text-display-lg:  clamp(2rem, 5vw, 3.75rem);   /* 32–60px */
  --text-display-md:  clamp(1.5rem, 3vw, 2.25rem); /* 24–36px */
}
```

## Display headings

```jsx
{/* display-2xl — Hero h1, Resume name */}
<h1 className="font-display font-bold text-display-2xl text-fg">
  Heading text
</h1>

{/* display-xl — Section heading (Work), Case Study title */}
<h2 className="font-display font-bold text-display-xl text-fg">
  Heading text
</h2>

{/* display-lg — CTA heading (About, Footer, Contact) */}
<h2 className="font-display font-bold text-display-lg text-fg">
  Heading text
</h2>

{/* display-md — Card / component heading (WorkEntry) */}
<h3 className="font-display font-bold text-display-md text-fg">
  Heading text
</h3>
```

The `font-display` + `font-bold` + `text-display-*` + `text-fg` pattern is standard for all headings. All four classes are always required together.

## Body copy

```jsx
{/* Body large — 18px — intro paragraphs, lead copy */}
<p className="font-body text-body-lg text-fg">Body copy here</p>

{/* Body — 16px — default body, case study content */}
<p className="font-body text-body text-fg">Body copy here</p>

{/* Body small — 14px — captions, metadata */}
<p className="font-body text-body-sm text-fg">Supporting copy here</p>

{/* Caption — 12px — minimum size */}
<p className="font-body text-caption text-fg-secondary">Caption or timestamp</p>
```

---

## Editorial (Ogg)

```jsx
{/* Regular — editorial accent, pull quote */}
<p className="font-editorial text-fg" style={{ fontSize: '32px', fontWeight: 400 }}>
  Pull quote or editorial accent text
</p>

{/* Italic — emphasis, pull quote accent */}
<p className="font-editorial text-fg" style={{ fontSize: '32px', fontWeight: 400, fontStyle: 'italic' }}>
  Italic editorial text
</p>
```

Font size for Ogg is always set inline — there is no fixed size token. Use the size that fits the layout context.

## Heading and body

Pair display tokens with body text one or two sizes below the visual weight of the heading. The larger the heading, the smaller the body text can afford to be.

<DoDont>
  <Do label="display-lg + body 16px — standard pairing">
    <div style={{ padding: '4px 0' }}>
      <p style={{ fontFamily: '"Cabinet Grotesk"', fontSize: 'clamp(2rem, 5vw, 3.75rem)', fontWeight: 700, lineHeight: 1.1, letterSpacing: '-0.03em', color: 'var(--fg)', margin: '0 0 10px' }}>Let's build something great</p>
      <p style={{ fontFamily: 'Areal, Arial, sans-serif', fontSize: '16px', lineHeight: 1.6, color: 'var(--fg)', opacity: 0.75, margin: 0 }}>Comfortable visual separation between headline and supporting copy.</p>
    </div>
  </Do>
  <Dont label="display-lg + 12px caption — gap too large">
    <div style={{ padding: '4px 0' }}>
      <p style={{ fontFamily: '"Cabinet Grotesk"', fontSize: 'clamp(2rem, 5vw, 3.75rem)', fontWeight: 700, lineHeight: 1.1, letterSpacing: '-0.03em', color: 'var(--fg)', margin: '0 0 10px' }}>Let's build something great</p>
      <p style={{ fontFamily: 'Areal, Arial, sans-serif', fontSize: '12px', lineHeight: 1.6, color: 'var(--fg)', opacity: 0.75, margin: 0 }}>Too large a jump. The body feels disconnected and hard to scan.</p>
    </div>
  </Dont>
</DoDont>

## Adjacent display sizes

Never stack two display tokens that are one step apart — the sizes are too close and neither leads clearly. Skip at least one step when pairing display text vertically.

## Ogg usage

Ogg belongs in editorial moments only — pull quotes, accent phrases, expressive breaks in layout. Never use Ogg Bold; it competes with Cabinet Grotesk. Regular and Italic are the only weights to use in the portfolio.

## Labels — color signals

Accent label color (`text-brand-primary`) signals a navigational or brand marker. Metadata labels use `text-fg-secondary` — they inform, not direct. Never use `text-brand-primary` on labels that don't carry a navigational or brand meaning — it reads as a link.
