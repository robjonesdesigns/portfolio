# Grid

The grid provides the foundation for visual rhythm and spatial consistency across every page and section. Every layout in the portfolio is built on a responsive column system that adapts across three breakpoints — from a 4-column mobile stack to a 12-column desktop layout.

## Base unit

The base unit is **4px** — the default step in Tailwind's spacing scale. All padding, margin, and gap values use values from this scale. Never use arbitrary spacing values.

## Grid fundamentals

The grid is built on the principle of dividing space by two. Column counts follow powers of two — 4, 8, 12 — adapting to the available screen width at each breakpoint.

### Columns and rows

Columns create the vertical key lines that all content aligns to. Row rhythm is established through consistent vertical spacing using the sizing scale. Content never fights the grid — it either spans a number of columns or aligns to a column edge.

Use the viewport toolbar (screen icon, top-right) to see the column grid adapt across all three breakpoints.

<Canvas of={GridStories.ColumnGrid} />

### Margins

Margins are the fixed space between the viewport edge and the content area. They grow with screen width to maintain visual balance as content scales.

  <thead>
    <tr><th>Breakpoint</th><th>Margin</th><th>Tailwind</th></tr>
  </thead>
  <tbody>
    <tr><td><code>base</code></td><td>24px</td><td><code>px-6</code></td></tr>
    <tr><td><code>md</code></td><td>40px</td><td><code>px-10</code></td></tr>
    <tr><td><code>lg</code></td><td>56px</td><td><code>px-14</code></td></tr>
  </tbody>

Margins are always applied through the `Container` component — never set manually on individual sections.

### Padding

Internal component padding is always a fixed multiple of the base unit. The standard value is **16px** (`p-4`) at all breakpoints. Always align type to the edge of box padding, not to the box boundary itself.

### Gutters

Gutters are the space between columns. They are fixed at each breakpoint — they do not scale fluidly between breakpoints.

Apply tighter gutters when content items are closely related. Use wider gutters when columns contain distinct, independent content areas.

## Grid behaviors

### Fluid columns

This portfolio uses fluid column structures throughout. Column widths scale as a percentage of the available grid area between breakpoints. At each breakpoint boundary, the column count steps up and the grid resets.

Multi-column layouts collapse to a single column at `base`. This ensures content is always readable regardless of screen size.

### Key lines

Consistent key lines are what make a layout feel ordered and intentional. The `Container` component creates the primary vertical key lines by ensuring all sections share the same left and right edges. No section breaks this alignment.

Horizontal key lines are established through the spacing scale — section dividers, borders, and vertical rhythm all snap to multiples of the base unit.

### Vertical rhythm

Section-level vertical spacing uses the sizing scale to create consistent breathing room between content areas. Standard section padding is `py-24 md:py-36` (96px mobile / 144px desktop). Tighter sections use `py-12 md:py-24`.

Never use arbitrary `padding-top` or `padding-bottom` values on sections. Always reference the spacing scale.

## Fit for purpose

Every page has a story that ends with a desired action. Layout choices should always start with content and user goals — not aesthetics.

Ask: what is the user trying to do on this page?

<table>
  <thead>
    <tr><th>Experience type</th><th>Layout approach</th></tr>
  </thead>
  <tbody>
    <tr><td><strong>Reading / long-form</strong></td><td>Narrow container (<code>sm</code>), single column, controlled line length</td></tr>
    <tr><td><strong>Scanning / browsing</strong></td><td>Wide container (<code>lg</code>), stacked entries with clear visual hierarchy</td></tr>
    <tr><td><strong>Structured reference</strong></td><td>Medium container (<code>md</code>), asymmetric columns, scannable sections</td></tr>
  </tbody>
</table>

This portfolio uses all three — case study prose uses `sm` for comfortable reading, the Work section uses `lg` with a stacked entry list for browsing, and the Resume uses `md` with an asymmetric grid for scannable structured content.

## Content hierarchy

Hierarchy helps users navigate complex content without getting lost. The grid is one of the primary tools for establishing hierarchy — not just through column count, but through the relationship between content areas.

### How our layouts communicate priority

- **Full-width sections** (Hero, Contact) — maximum visual weight, single focused message
- **2-column equal** (About) — headshot and bio/skills; neither side dominates
- **Asymmetric 2-column** (Resume) — 1fr meta + 2fr content; label column anchors the detail column
- **Stacked single-column** (Work) — sequential WorkEntry items; vertical flow drives scanning
- **Narrow single-column** (Case study body) — long-form reading; line length controlled for comfort

### Spacing reinforces structure

Vertical rhythm between sections uses consistent spacing tokens. Tighter gaps group related content; looser gaps separate sections. Never use arbitrary spacing — always reference the spacing scale.

## Continuity

Pattern recognition across pages reduces cognitive load. Users build a mental model of the layout early — breaking it mid-journey creates friction.

### Shared alignments

Every section in this portfolio shares the same horizontal rhythm through the `Container` component. The left edge of body text, headings, and labels all align to the same invisible column — whether on the homepage, resume, or a case study.

### Consistent section structure

Each page section follows the same scaffolding pattern:

```
section (owns vertical padding)
└── Container (owns max-width + horizontal padding)
    └── content
```

This means adding a new section never requires thinking about margins — the Container handles it automatically.

## Gutter guidance

Gutters are the space between columns. The right gutter size depends on the relationship between the content items — are they related or independent?

<table>
  <thead>
    <tr><th>Gutter</th><th>Token</th><th>When to use</th></tr>
  </thead>
  <tbody>
    <tr><td><strong>Tight</strong></td><td><code>gap-6</code> · 24px</td><td>Items in the same content group — metadata columns, skill badges</td></tr>
    <tr><td><strong>Default</strong></td><td><code>gap-8</code> · 32px</td><td>Standard card grids — project cards, work entries</td></tr>
    <tr><td><strong>Loose</strong></td><td><code>gap-12 md:gap-20</code></td><td>Two distinct content areas — About section left/right split</td></tr>
  </tbody>
</table>

Larger gutters signal independence between columns. Smaller gutters signal relationship. Never use arbitrary gap values — always use the spacing scale.

## Layout patterns

Three layout patterns cover all multi-column arrangements in the portfolio.

### 2-column equal — About

Headshot on the left, bio and skills on the right. Splits into two equal columns at `md` (768px), stacks on mobile. The loose gutter signals that the two sides are distinct but related.

```
grid md:grid-cols-2 gap-12 md:gap-20
```

A narrow label column (1fr) paired with a wider content column (2fr). Used for every `ExperienceBlock` on the Resume — company/role/period on the left, bullet points on the right. The asymmetry creates a clear anchor for scanning.

## Documentation

This portfolio's grid is built on **Tailwind CSS Grid** — native CSS Grid with responsive utility classes. The `Container` component is the primary layout primitive that handles max-width and horizontal padding. Multi-column layouts are composed with Tailwind's `grid` utilities directly in JSX.

No separate grid package is needed. Everything is available through Tailwind's responsive prefix system.

## CSS Grid

Multi-column layouts use Tailwind's `grid` utilities. Column count and gutter are always defined with responsive prefixes so layouts collapse gracefully on mobile.

### 2-column equal — About

```jsx
<Container>
  <div className="grid md:grid-cols-2 gap-12 md:gap-20">
    <div>{/* left */}</div>
    <div>{/* right */}</div>
  </div>
</Container>
```

### Asymmetric 2-column — Resume

The `[1fr_2fr]` syntax sets a narrow label column alongside a wider content column. This is a Tailwind arbitrary value — use bracket notation for non-standard column templates.

```jsx
<div className="grid md:grid-cols-[1fr_2fr] gap-4 md:gap-12">
  <div>{/* meta: company, role, period */}</div>
  <div>{/* content: bullet points */}</div>
</div>
```

### Stacked list — Work

No grid needed. Work entries are block-level and stack naturally.

```jsx
<Container>
  <div>
    {projects.map(p => (
      <WorkEntry key={p.slug} project={p} />
    ))}
  </div>
</Container>
```

## Responsive utilities

Tailwind's breakpoint prefixes are the equivalent of CSS breakpoint mixins. Apply them to any utility class to change behavior at a given screen width.

<table>
  <thead>
    <tr><th>Prefix</th><th>Min-width</th><th>Equivalent</th></tr>
  </thead>
  <tbody>
    <tr><td><em>(none)</em></td><td>0px</td><td>Mobile-first base styles</td></tr>
    <tr><td><code>md:</code></td><td>768px</td><td>Tablet and above</td></tr>
    <tr><td><code>lg:</code></td><td>1024px</td><td>Desktop and above</td></tr>
  </tbody>

### Show / hide at breakpoint

```jsx
{/* Visible on mobile only */}
<div className="block md:hidden">…</div>

{/* Visible on tablet and above only */}
<div className="hidden md:block">…</div>

{/* Visible on desktop only */}
<div className="hidden lg:block">…</div>
```

### Responsive column count

```jsx
{/* 1 col → 2 col at md → 3 col at lg */}
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
  …
</div>
```

### Responsive spacing

```jsx
{/* Tighter gap on mobile, wider on desktop */}
<div className="grid md:grid-cols-2 gap-6 md:gap-12 lg:gap-20">
  …
</div>
```

## Aspect ratio

When sizing image or media boxes, use Tailwind's `aspect-*` utilities to enforce consistent proportions. This prevents layout shift and keeps grids visually stable.

```jsx
{/* Portrait headshot — About section */}
<div className="aspect-[3/4] overflow-hidden rounded-2xl">
  <img src="..." className="w-full h-full object-cover" />
</div>

{/* 16:9 video embed — case study process media */}
<div className="aspect-video overflow-hidden rounded-xl">
  <video src="..." autoPlay muted loop playsInline />
</div>

{/* Square — icons, thumbnails */}
<div className="aspect-square">…</div>
```
