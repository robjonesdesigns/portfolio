## Introduction

Spacing is the negative area between elements and components. It is controlled in code through margin and padding. The portfolio offers a single spacing scale as CSS custom properties and as Tailwind utilities — values are identical, use whichever fits the context.

The spacing scale complements the layout grid and type scale by using multiples of 4px. It includes small increments for detail-level design and larger increments for layout density and section rhythm.

---

## Applying spacing

The spacing scale can be applied to `margin`, `padding`, `gap`, and related properties. Replace `*` in the Tailwind column with any spacing prefix — `p`, `m`, `mt`, `px`, `gap`, etc.

```jsx
// Padding
<div className="p-4">…</div>       // 16px
<div className="px-6 py-4">…</div> // 24px horizontal, 16px vertical

// Margin
<div className="mt-12">…</div>     // 48px top margin

// Gap
<div className="flex gap-6">…</div> // 24px between flex children
```

For spacing between major page sections, use the larger end of the scale (`space-4xl` at 96px or `space-5xl` at 128px). For spacing inside components, use the smaller end (`space-xs` through `space-lg`).

## Designing with space

Every part of a UI should be intentional — including the empty space between elements. The amount of space between items creates relationships and hierarchy.

**Creating relationships**

Elements near each other are perceived as meaningfully related. As more space is added between elements, their perceived relationship weakens. Use spacing to group associated information without needing lines or other graphical dividers.

**Creating hierarchy**

Elements with more space around them tend to be perceived as more important than elements with less space. Section headers have more surrounding space, giving them focus. As headings descend in importance they receive less space.

**White space**

Empty space helps with information processing. Too much dense information can be disorienting. Sections of a UI can be dense, but the whole page should have enough breathing room for the user's eye to rest.

## CSS custom properties

Access spacing tokens directly as CSS custom properties in inline styles or `globals.css`.

```css
/* In globals.css */
.my-component {
  padding: var(--space-lg);         /* 24px */
  margin-bottom: var(--space-2xl);  /* 48px */
  gap: var(--space-md);             /* 16px */
}
```

```jsx
// In JSX inline styles
<div style={{ padding: 'var(--space-lg)', gap: 'var(--space-md)' }}>
  …
</div>
```

## Component example

A card using spacing tokens throughout.

```jsx
function ProjectCard({ title, description }) {
  return (
    // p-6 = space-lg = 24px padding
    <div className="bg-surface border border-border rounded-2xl p-6">

      {/* gap-2 = space-sm = 8px between label and heading */}
      <div className="flex flex-col gap-2 mb-4">
        <p className="font-body font-bold text-label text-brand-primary">
          Case Study
        </p>
        <h3 className="font-display font-bold text-display-md text-fg">
          {title}
        </h3>
      </div>

      {/* mb-6 = space-lg = 24px below description */}
      <p className="font-body text-body text-fg-secondary mb-6">
        {description}
      </p>

      <a className="text-brand-primary text-sm font-medium">
        View project →
      </a>
    </div>
  )
}
```

## Section spacing

For vertical rhythm between major page sections, use the large end of the scale.

```jsx
// py-24 = space-4xl = 96px — standard section vertical padding
<section className="py-24">
  <Container>…</Container>
</section>

// gap-16 = space-3xl = 64px — between WorkEntry cards
<div className="flex flex-col gap-16">
  {projects.map(p => <WorkEntry key={p.slug} {...p} />)}
</div>
```
