# Themes

Themes are used to customize the portfolio's visual style to reflect a specific aesthetic. By using role-based tokens, all components update automatically when the theme changes — no individual overrides needed.

## Theming basics

Themes are used to modify the visual style of all components by changing a set of universal token values. With tokens, the code only needs to change in one place to see the effect system-wide.

<table>
  <thead>
    <tr><th>Term</th><th>Definition</th></tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Theme</strong></td>
      <td>A collection of visual attributes assigned to the tokens in order to create a specific aesthetic. The portfolio has two themes: Light and Dark.</td>
    </tr>
    <tr>
      <td><strong>Token</strong></td>
      <td>A role-based identifier that assigns a value to a theme. Tokens are universal and never change across themes — only their values change.</td>
    </tr>
    <tr>
      <td><strong>Role</strong></td>
      <td>The systematic usage of a token. Roles cannot change between themes — <code>--fg</code> is always foreground text in both light and dark.</td>
    </tr>
    <tr>
      <td><strong>Value</strong></td>
      <td>The actual color (hex code or rgba value) assigned to a token in a given theme. For example, <code>--fg</code> is <code>#222222</code> in light and <code>#fffbf5</code> in dark.</td>
    </tr>
  </tbody>
</table>

## Tokens

With tokens, a single value change propagates system-wide. Tokens are used across all components and keep global patterns consistent. There are four token categories:

<table>
  <thead>
    <tr><th>Category</th><th>Tokens</th><th>Applied to</th></tr>
  </thead>
  <tbody>
    <tr><td><strong>Color</strong></td><td><code>--bg</code>, <code>--fg</code>, <code>--accent</code>, <code>--on-accent</code>, <code>--surface</code>, <code>--border</code>, <code>--fg-secondary</code>, <code>--brand-hover</code></td><td>Backgrounds, text, brand, dividers</td></tr>
    <tr><td><strong>Spacing</strong></td><td><code>--space-xs</code> → <code>--space-5xl</code></td><td>Margin, padding, gap — same values across both themes</td></tr>
    <tr><td><strong>Typography</strong></td><td><code>--text-display-{'*'}</code></td><td>Fluid display sizes — same values across both themes</td></tr>
    <tr><td><strong>Motion</strong></td><td><code>--duration-{'*'}</code></td><td>Transition durations — same values across both themes</td></tr>
  </tbody>
</table>

Only color tokens change between themes. Spacing, typography, and motion tokens are invariant.

For the full list of color tokens and their values in each theme, see the [Color — Tokens](/docs/elements-color--docs) tab.

## System preferences

Most operating systems (macOS, iOS, Windows, Android) expose the user's preferred color scheme via `prefers-color-scheme`. The `useTheme` hook reads this preference on first load and applies the matching theme automatically.

Where no saved preference exists, the system preference is used as the default. Users can override it at any time using the `ThemeToggle` component in the navigation bar.

## CSS custom properties

The two themes are defined in `globals.css` using the `.dark` class selector. When `.dark` is on `<html>`, all adaptive tokens resolve to their dark values simultaneously.

```css
/* globals.css */

/* Light theme — default (no class needed) */
:root {
  --bg:           #fffbf5;
  --fg:           #222222;
  --fg-secondary: rgba(34, 34, 34, 0.55);
  --accent:       #813746;
  --on-accent:    #fffbf5;
  --surface:      #f7f3f5;
  --border:       rgba(34, 34, 34, 0.10);
  --brand-hover:  #612935;
}

/* Dark theme — applied when .dark is on <html> */
.dark {
  --bg:           #1c1a16;
  --fg:           #fffbf5;
  --fg-secondary: rgba(255, 251, 245, 0.55);
  --accent:       #e36f86;
  --on-accent:    #1c1a16;
  --surface:      #252220;
  --border:       rgba(255, 251, 245, 0.10);
  --brand-hover:  #c15e72;
}
```

## useTheme hook

The `useTheme` hook manages the active theme. It reads the saved preference from `localStorage`, falls back to the OS `prefers-color-scheme`, and applies `.dark` to `<html>` accordingly.

```js
// src/hooks/useTheme.js
export function useTheme() {
  const [dark, setDark] = useState(() => {
    const saved = localStorage.getItem('theme')
    if (saved) return saved === 'dark'
    return window.matchMedia('(prefers-color-scheme: dark)').matches
  })

  useEffect(() => {
    document.documentElement.classList.toggle('dark', dark)
    localStorage.setItem('theme', dark ? 'dark' : 'light')
  }, [dark])

  return { dark, toggle: () => setDark(d => !d) }
}
```

## ThemeToggle component

The `ThemeToggle` component calls `useTheme` and renders a button that switches between themes.

```jsx
// Usage — placed in Navbar
import { useTheme } from '../hooks/useTheme'

function Navbar() {
  const { dark, toggle } = useTheme()
  return (
    <nav>
      {/* ... */}
      <ThemeToggle dark={dark} onToggle={toggle} />
    </nav>
  )
}
```

## System preference detection

Detect the OS preference without `useTheme` — useful in non-React contexts or for SSR hydration.

```js
// Detect on first load
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
if (prefersDark) document.documentElement.classList.add('dark')

// Listen for OS-level changes
window.matchMedia('(prefers-color-scheme: dark)')
  .addEventListener('change', e => {
    document.documentElement.classList.toggle('dark', e.matches)
  })
```

## Inline theming

For Storybook stories where you need to lock a specific background regardless of the active theme, use static Tailwind tokens rather than the adaptive ones. These values are fixed — they never switch.

```jsx
// Story decorator — force light background
export const decorators = [
  (Story) => (
    <div className="bg-cream p-8">
      <Story />
    </div>
  ),
]

// Story decorator — force dark background
export const decorators = [
  (Story) => (
    <div className="bg-dark-bg p-8">
      <Story />
    </div>
  ),
]
```
